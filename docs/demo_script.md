# Demo 演示脚本（组员 B 用）

## 打开方式

浏览器输入：
```
http://localhost:8501
```
（或老师提供的外网地址）

---

## 第 1 步：展示 GitHub 仓库查询（30 秒）

### 输入（粘贴到对话框）
```
你能查看我的GitHub仓库吗 https://github.com/yfn-1116
```

### 预期结果
列出约 4-8 个公开仓库：career-agent-assistant、chem-auto-titration、polyu-internship-project、opencode 等，每个带项目名和描述。

### 讲解词

"第一个功能——**GitHub 仓库实时查询**。

用户只需要在聊天框里粘贴一个 GitHub 链接，系统会自动识别这是一个仓库查询请求。

它做了什么？调用 GitHub 工具，通过 HTTP 直接访问 GitHub 服务器，拉取这个用户名下的所有公开仓库列表，包括项目名、编程语言、Star 数、项目描述。不需要登录，不需要 Token。

背后的技术：系统内部有一个 GitHub 工具，通过 urllib 直接调 GitHub 的 REST API。如果 API 限流了，会自动回退到本地的知识库缓存——你之前索引过的仓库数据不会丢。

这个功能有什么用？求职的时候，你的 GitHub 仓库就是最好的简历。系统能自动了解你做过哪些项目，后面做 JD 匹配的时候，这些项目就会被用作'证据'。"

---

## 第 2 步：查看具体仓库内容（30 秒）

### 输入
```
https://github.com/yfn-1116/career-agent-assistant这个仓库做什么的
```

### 预期结果
返回该仓库的完整 README 内容：项目定位（是不是 GPT 包装壳、是不是自动投递机器人）、核心能力模块表格、技术栈、快速运行命令。

### 讲解词

"第二个功能——**读取具体仓库的 README**。

粘贴具体仓库链接，系统去 raw.githubusercontent.com 抓取这个仓库的 README 文件，把项目定位、技术栈、运行方式全部展示出来。

可以看到这个项目叫 career-agent-assistant，就是一个求职 Agent 系统。它的 README 里写清楚了——这不是 GPT 包装壳、不是自动投递机器人，而是一个有 Evidence Gate 证据约束的 Agent。

这个功能的关键在于——**系统不只看到仓库名，而是能读懂里面的内容**。后面做 JD 匹配的时候，它知道这个项目具体做了什么、用了什么技术、实现了什么功能。"

---

## 第 3 步：查看知识库中的项目经历（20 秒）

### 输入
```
我有哪些项目经历？
```

### 预期结果
列出知识库中索引的所有项目：智能投递辅助 Agent、自动滴定系统、PolyU Smart Journey、CNN 狗品种分类等，每条带技术栈和简要描述。

### 讲解词

"第三个功能——**个人知识库检索**。

系统维护了一个个人知识库，把简历、项目经历、技能描述、GitHub 仓库内容全部索引在里面。问一句话就能检索出所有相关经历。

那知识库的数据怎么来的？两种方式——第一，用户上传 PDF 简历或 Markdown 文件，系统自动分块、索引。第二，刚才查过的 GitHub 仓库内容，也会缓存到知识库里。

这些数据不会丢——知识库用 JSONL 文件持久化存储，服务器重启也不影响。而且支持 BM25 关键词检索，不管你怎么问，都能找到相关内容。"

---

## 第 4 步：JD 匹配分析（核心功能，1.5 分钟）

### 输入（粘贴以下完整 JD）
```
Agent开发实习生 (140-190元/天)
工作地点：深圳
实习要求：5天/周，最少3个月
最低学历：本科在读
技术标签：Java、C++、Python
职位描述
希望有 AI 相关的开发经验
岗位职责：
负责 AI Agent 系统的开发和优化，包括记忆模块、规划模块和工具调用模块；
开发和完善基于大语言模型（LLM）的智能体系统，提升 Agent 的自主决策和任务执行能力；
设计并实现 Agent 的工具使用接口，实现与各类外部系统和 API 的集成。
优化 Agent 的对话系统，提升用户交互体验和任务完成效率。
开发 Agent 的知识库管理系统，实现高效的信息存储和检索。
任职要求：
本科及以上学位在读，计算机科学、人工智能或相关专业优先；
熟悉 Python、Java 等主流编程语言，具备良好的代码风格和工程实践；
熟悉大语言模型（LLM）的原理和应用，有 GPT 等模型应用经验；熟悉机器学习、深度学习等 AI 技术，了解最新研究进展；
具备优秀的问题解决能力和系统架构设计能力；
具备良好的团队协作能力和沟通能力。
```

### 等待
15-20 秒（Qwen 大模型在做匹配分析，左下角会显示"正在思考..."）

### 预期结果
返回完整的匹配分析报告，包含：

**匹配度：85%**
**强项**：
- Python：全部项目以 Python 为主栈
- LLM 原理和应用：RAG 匹配 Agent 体现 Prompt Engineering、function calling、模型路由
- AI Agent 系统开发：多 Agent 素材生产系统含工具调用、规划模块、记忆模块
- 知识库管理系统：完整构建 Markdown→Chroma→LangChain→Streamlit 全链路

**弱项**：
- Java：无相关项目经历
- 机器学习深度学习：仅课程级别，非优势项

**简历要点（STAR 法则，3 条）**：
每条标注证据来源，如"证据：projects.md '智能投递辅助 Agent'"

**HR 沟通话术**：
一段基于真实经历生成的打招呼文本

### 讲解词

"这是最核心的功能——**JD 智能匹配分析**。

刚才展示了三个功能——查 GitHub、读仓库、搜知识库。现在把它们串起来。

粘贴一段完整的岗位描述，系统做了什么事？

**第一步，感知**——系统识别出这是一段 JD 文本，自动路由到岗位分析流程。

**第二步，检索**——从知识库里搜索所有跟这个 JD 相关的经历。JD 要求 Agent 开发、工具调用、知识库管理——系统在知识库里找到 SmartApplyAgent 项目覆盖了这些要求。

**第三步，分析**——把所有检索到的证据和 JD 一起送给 Qwen 大模型。大模型不是简单地做关键词匹配——它理解 JD 的语义，对比每个项目具体做了什么，然后告诉你哪些要求被覆盖了、哪些没有。

**第四步，生成**——大模型基于证据生成简历要点。注意看，用的是 STAR 法则——情境、任务、行动、结果。每条都标注了证据来源。它不会编造——如果说'熟悉 Java'但知识库里没有 Java 项目，它会诚实告诉你'建议补充'。

**第五步，话术**——最后还生成了一段 HR 沟通话术，基于你的真实项目经历，不是千篇一律的模板。

整个过程，系统遵循一条核心原则：**所有输出必须基于证据，不得编造**。这就是我们这个项目叫 Evidence-Constrained Agent 的原因。"

---

## 第 5 步：展示 HR 沟通话术，总结收尾（30 秒）

### 讲解词（指着第 4 步输出的最后一段）

"最后这一部分是 HR 沟通话术。基于真实项目经历生成——提到了 RAG 求职匹配系统、多 Agent 素材生产系统、具体的技能匹配点。不是'我对这个岗位很感兴趣'这种模板。"

---

### 总结

"总结一下，我们这个系统做了三件事：

1. **GitHub 仓库实时查询**——贴链接就能看项目
2. **个人知识库检索**——简历、项目、技能全部索引，随时查
3. **JD 智能匹配分析**——输入岗位需求，输出匹配度 + STAR 简历要点 + HR 话术

核心原则：所有输出基于真实证据，不编造经历。这就是 Evidence-Constrained Agent。"
