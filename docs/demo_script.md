## 演示 1：GitHub 仓库实时查询

### 问什么
```
你能查看我的GitHub仓库吗 https://github.com/yfn-1116
```

### 系统回答什么
列出约 4-8 个公开仓库：career-agent-assistant、chem-auto-titration、polyu-internship-project、opencode 等，每个带项目名和描述。

### 这说明了我们做了什么
"我们做了一个**能实时访问 GitHub 的工具**。用户粘贴一个链接，系统通过 HTTP 直接调 GitHub API，拉取公开仓库列表。不需要登录，不需要 Token。如果 API 限流了，自动回退本地缓存。

这不是写死的演示数据——你换任何一个 GitHub 用户名，它都能查。"

## 演示 2：读取具体仓库内容

### 问什么
```
https://github.com/yfn-1116/career-agent-assistant这个仓库做什么的
```

### 系统回答什么
返回该仓库的完整 README：项目定位、核心能力、技术栈、快速运行命令。

### 这说明了我们做了什么
"我们做了一个**能读懂仓库内容的工具**。它去 raw.githubusercontent.com 抓取 README 原文，不是只看仓库名——而是理解里面写了什么、用了什么技术、实现了什么功能。

这些内容会被缓存到知识库，后面做 JD 匹配的时候，就是分析这些真实内容的依据。"

## 演示 4：JD 智能匹配分析（核心）

### 问什么（粘贴完整 JD）
```
Agent开发实习生 (140-190元/天)
工作地点：深圳
实习要求：5天/周，最少3个月
最低学历：本科在读
技术标签：Java、C++、Python
职位描述
希望有 AI 相关的开发经验
岗位职责：
负责 AI Agent 系统的开发和优化，包括记忆模块、规划模块和工具调用模块；
开发和完善基于大语言模型（LLM）的智能体系统，提升 Agent 的自主决策和任务执行能力；
设计并实现 Agent 的工具使用接口，实现与各类外部系统和 API 的集成。
优化 Agent 的对话系统，提升用户交互体验和任务完成效率。
开发 Agent 的知识库管理系统，实现高效的信息存储和检索。
任职要求：
本科及以上学位在读，计算机科学、人工智能或相关专业优先；
熟悉 Python、Java 等主流编程语言，具备良好的代码风格和工程实践；
熟悉大语言模型（LLM）的原理和应用，有 GPT 等模型应用经验；熟悉机器学习、深度学习等 AI 技术，了解最新研究进展；
具备优秀的问题解决能力和系统架构设计能力；
具备良好的团队协作能力和沟通能力。
```

### 等待
15-20 秒（左下角显示"正在思考..."）

### 系统回答什么

**匹配度：85%**

**强项**（逐条说明哪个项目覆盖了哪个要求）：
- Python → 全部项目 Python 主栈
- LLM 原理和应用 → RAG Agent 体现 Prompt Engineering、function calling
- AI Agent 系统开发 → 多 Agent 系统含工具调用、规划、记忆模块
- 知识库管理系统 → Chroma + LangChain 全链路

**弱项**：
- Java → 无相关项目经历
- 机器学习 → 仅课程级别

**简历要点（STAR 法则，3 条）**：
每条标注证据来源，如"证据：projects.md '智能投递辅助 Agent'"

**HR 沟通话术**：
基于真实项目经历生成的打招呼文本

### 这说明了我们做了什么
"这是我们最核心的功能——**JD 智能匹配分析**。把前面三个功能串起来了。

系统做了五步：

**第一步，感知**——识别输入是 JD 文本，自动路由到岗位分析流程。

**第二步，检索**——从知识库里搜索所有跟这个 JD 相关的经历。JD 要求 Agent 开发、工具调用、知识库管理——系统找到了 SmartApplyAgent 覆盖了这些。

**第三步，分析**——把所有证据和 JD 一起送给 Qwen 大模型。大模型理解语义，对比每个项目做了什么，告诉你哪些要求被覆盖、哪些没有。

**第四步，生成**——基于证据生成简历要点。用 STAR 法则——情境、任务、行动、结果。每条标注证据来源。它不会编造——知识库里没有 Java 项目，它就诚实告诉你'建议补充'。

**第五步，话术**——基于真实经历生成 HR 沟通文本，不是模板。

整个过程遵循一个原则：**所有输出必须基于证据，不得编造**。这就是 Evidence-Constrained Agent。"
