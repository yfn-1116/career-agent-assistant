# 本地到服务器差异

## 问题描述

项目可能在本地台式机能运行，但在学校服务器因为 Python 版本、依赖、网络、端口或文件路径不同而失败。

## 风险影响

- demo 无法复现。
- 模型调用失败。
- Streamlit 页面无法访问。
- 文件路径错误导致资料加载失败。

## 初步解决策略

- 优先 CLI demo。
- 记录本地开发和服务器部署 runbook。
- 使用相对路径和样例数据。
- 准备 mock / rule-based 输出兜底。

## 第一阶段处理方式

先保证本地 CLI demo 可运行，再补学校服务器部署文档。Streamlit 作为轻量展示，不作为唯一演示路径。

## 后续优化方向

- 补充依赖锁定。
- 增加环境检查脚本。
- 增加部署故障排查清单。

## 后续维护规则

服务器问题必须同步记录到 `documents/98-runbook/04-troubleshooting.md`。
