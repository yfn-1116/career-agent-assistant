# Demo 稳定性

## 问题描述

Demo 当天可能出现模型不可用、网络失败、服务器异常或样例输出不稳定。

## 风险影响

- 无法展示核心能力。
- 演示结果和讲解不一致。
- 项目可信度下降。

## 初步解决策略

- 准备固定示例资料和示例 JD。
- 准备 CLI demo 作为最稳定路径。
- 保存预期 Markdown 输出用于兜底展示。
- 模型调用失败时使用 mock / rule-based 输出。

## 第一阶段处理方式

第一阶段 demo 必须可在本地通过固定输入复现。Streamlit 只作为展示增强，不替代 CLI。

## 后续优化方向

- 增加 demo 自检命令。
- 增加模型不可用时的降级模式。
- 增加服务器部署 smoke test。

## 后续维护规则

每个 demo 任务必须说明输入、预期输出和失败兜底。
