# 最终展示脚本（5-8 分钟）

> 适用场景：期末答辩、实习面试展示
> 风格：口语化，照着讲即可

---

## 开场（30 秒）

大家好，我做的项目叫 career-agent-assistant，是一个基于 RAG 和多 Agent 的实习投递智能辅助工具。

## 痛点（30 秒）

找实习的时候，我相信大家都有这个体验——对着 10 份 JD，你需要反复翻自己的简历和项目文档，去回忆"我有没有做过这个""我能不能对上这个要求"。效率很低，而且容易遗漏有价值的经历。

所以我想做一个工具：把我的技能、项目经历、实习经历写成结构化的 Markdown 文档，然后让 AI 帮我自动匹配 JD。

## 项目目标（30 秒）

需要强调一下，这个工具**不是自动投递机器人**，它的定位是辅助分析和生成——帮你理解自己和岗位的匹配度，告诉你哪里强、哪里弱、怎么改简历、怎么联系 HR。

## 技术路线（1 分钟）

整体架构分三层：

- **RAG 层**：把用户 Markdown 资料加载、分块、向量化、建索引，然后用 JD 关键词去检索最相关的经历片段
- **Agent 层**：四个 Agent 各司其职——JD 解析 Agent 提取岗位要求，检索 Agent 调用 RAG pipeline 找证据，分析 Agent 做匹配对比，生成 Agent 输出简历 bullet 和沟通话术
- **Workflow 层**：把这四个 Agent 串起来，共享一个 AgentTaskState，每个 Agent 只读写自己的字段

当前阶段全部用规则和模板实现，不调大模型，保证可复现。但已经预留了 ModelProvider 接口，后续接入 DeepSeek 只需要一行配置。

另外还做了一个轻量评估模块，可以对多个 JD 跑批量评估，看看输出质量怎么样。

## 工程实践（30 秒）

整个项目有 216 个自动化测试，覆盖了 RAG、Agent、Workflow、Demo、ModelProvider、Evaluation 全栈。开发流程是文档先行——先写设计文档和技术决策，再写任务卡，再写代码。每个任务独立 commit，20 多个 commit 历史清晰可追溯。

## 演示（2 分钟）

（切换到终端）

先跑一下测试，确认环境正常（运行 pytest）。

然后跑 CLI demo：

```bash
PYTHONPATH=src python demo/cli/run_job_match_demo.py
```

看输出文件——这是完整的分析报告：JD 解析、检索到的证据、匹配分析、简历 bullet、沟通话术。

再跑一下 Streamlit 版本（启动 streamlit），可以看到可视化的界面，选择 JD、点运行、查看结果。

最后跑一下评估：

```bash
PYTHONPATH=src python demo/evaluation/run_evaluation.py
```

可以看到 4 个 JD 的评估分数，平均 0.9 左右。

## 总结（1 分钟）

这个项目的价值在于：

1. 完整的 RAG + Multi-Agent 闭环，可以端到端运行
2. 严格的证据溯源，输出不编造经历
3. 零外部依赖，不联网也能跑
4. 工程规范扎实，216 测试 + 30+ 份文档

## 局限和后续（30 秒）

坦诚说一下不足：

- 检索目前是关键词匹配，不是语义检索，后续会换 Embedding
- 分析和生成是规则模板，不是 LLM
- 没有自动搜索岗位功能

后续最想做的：接入 DeepSeek API 和 Embedding 检索，效果会有质的提升。
