# 需求文档目录

## 用途

本目录记录第一阶段需求边界，包括用户场景、MVP 范围、非目标和验收标准。它回答“为什么做、给谁做、做到什么程度算完成”。

## 应放内容

- 大学生实习求职中的真实使用场景。
- 第一阶段必须跑通的核心能力。
- 明确不做的功能，防止项目膨胀。
- 可人工检查或命令验证的验收条件。

## 当前状态

Phase 1 已补强第一阶段需求边界。当前需求聚焦 RAG + 多 Agent 原型，不覆盖完整求职平台。

## 后续维护规则

- 需求文档只描述目标、边界和验收，不写具体代码结构。
- 新增需求必须说明属于 MVP、后续阶段还是明确非目标。
- 需求变更后必须同步检查 `documents/02-design/` 和 `documents/99-project-planning.md`。
